# Flatpak packaging

A Flatpak build of Aldur Appraiser for Linux (the recommended way to run it on
Wayland desktops — screen capture goes through the desktop ScreenCast portal,
which is exactly how Flatpak's sandbox grants screen access).

## Why this layout

- **Runtime:** `org.kde.Platform//6.9` (Qt6 + GStreamer + `pipewiresrc` +
  Python 3.12). 6.9 is deliberate: the newer 25.08 runtimes ship Python 3.13,
  which has no `onnxruntime`/`rapidocr` wheels yet.
- **PySide6:** from `io.qt.PySide.BaseApp//6.9` (built against the runtime),
  not the giant PySide6 pip wheel.
- **`gi` / GStreamer / `pipewiresrc`:** provided by the KDE runtime — no
  PyGObject module needed, so the portal capture backend works out of the box.
- **Other Python deps:** pinned offline in `python-deps.yaml`
  (generated by `flatpak-pip-generator`, all prebuilt wheels — no source builds).
- **The app itself:** installed from a pre-built universal (`py3-none-any`)
  wheel, so the offline build sandbox never needs a Python build backend.
  Bundled data (detection template, icon, `config.toml`) is installed under
  `/app/share/aldur-appraiser` and found via the `ALDUR_RESOURCE_DIR` env var.

## Build

```bash
flatpak install flathub org.kde.Platform//6.9 org.kde.Sdk//6.9 io.qt.PySide.BaseApp//6.9
./build.sh
flatpak run io.github.t0nd3.AldurAppraiser
```

`build.sh` rebuilds the wheel and runs `flatpak-builder`. To regenerate pinned
deps after editing `requirements.txt`, see the comment at the top of `build.sh`.

> `flatpak-pip-generator` is fetched on demand and is gitignored; grab it from
> [flatpak-builder-tools](https://github.com/flatpak/flatpak-builder-tools).
